二、 基于FPGA的高性能离子电子皮肤感知系统

系统设计作者：李志斌（硬件开发和Verilog设计），张承璧（仿真验证，文档书写）

1. 设计概述|Design Introduction

1.1 1.1 设计目的

随着智能机器人、可穿戴设备和电子皮肤技术的快速发展，触觉感知已成为人机交互中的关键环节。通过触觉，机器和设备能够像人类一样感知压力、温度及物体表面的微小变化。离子电子皮肤利用电双层效应（Electrical Double Layer, EDL），展现出超高的灵敏度、抗噪声能力以及对静态和动态压力的快速响应，特别是在柔性和高分辨率感知方面具有显著优势。相比于传统触觉传感器，离子电子皮肤能够提供更加精确的触觉信息，并能够适应复杂的应用场景。然而，要实现多通道高精度的触觉信号处理，系统需要满足更高的性能要求。FPGA凭借其强大的并行处理能力，成为应对这一挑战的理想解决方案。本设计通过FPGA构建多通道高性能离子电子皮肤感知系统，实现高采样率和低串扰的信号采集与实时处理，以满足复杂场景下的触觉感知需求。

与传统的基于时分复用的传感器设计[2]相比，我们提出了一种创新的频分复用架构，使得不同传感器通道可以同时工作，避免了数据离散问题。通过将相敏检测算法（Phase-Sensitive Detection, PSD）部署在FPGA上，我们充分发挥了FPGA的并行计算优势，大幅提升了系统的处理速度和精度。与微控制器或数字信号处理器（DSP）架构相比，FPGA能够在更短的时间内完成多通道数据解调，极大地提高了系统的动态响应能力。

基于该设计，我们构建了一套基于离子电子皮肤的触觉感知系统，将触觉响应的采集与解调全部集成于FPGA中，实现了理论上最快的数据处理速率。在实验中，我们通过弹力球模拟碰撞，传感器阵列成功感知落点和压力，并通过压力大小预测弹起高度，从而验证了系统的高精度和快速响应能力。我们还对比了在ARM架构的微处理器上实现相同解调流程的性能差异，进一步证实了FPGA架构在处理速度和并行处理方面的显著优势。

1.2 应用领域

本设计在多个应用领域展现出广泛的前景。通过FPGA的多通道并行处理能力，经过实际场景中的动态刺激感知验证，证明了其在离子电子皮肤感知系统中的传感接口设计具有显著的优势。该系统的高效数据处理与精准感知能力对开发高性能柔性电子皮肤具有重要意义。在物联网、智能机器人、智能可穿戴监测等领域，尤其在人机交互和智慧医疗等应用中，柔性电子皮肤将成为不可或缺的技术支撑。

1.3 主要技术特点

（1） 多路并行相敏验波算法： 在FPGA上实现了多路并行的相敏验波算法，主要包括正交参考信号生成模块、乘法器模块、平方和运算模块以及低通滤波器模块。输入信号通过Verilog代码驱动的多通道数模转换芯片进行采样。该算法基于互相关检测原理，通过已知频率的参考信号与输入信号的互相关检测获得特定频率信号的幅值，避免了不同通道之间的信号干扰，从而实现了多通道的并行读取，大幅提升了传感器的读出速率。系统还搭建了多通道数字信号发生器，用于产生不同频率的正弦波以调制传感器的电容响应。

（2） 数据采集处理 根据数据采集速率及与上位机通信的需求，设计了数据缓存模块和内存读写模块，构建了从数据采集、解调到进一步处理的通路，使得系统使用更加便捷，实用性更强。在本项目中，我们搭建了运行于操作系统上的上位机，通过USB协议与板上系统通信，完成了触觉信息的三维可视化展示。

（3） 算法的硬件实现对比软件实现 在ARM架构的微型处理器上，我们编写了相同算法的C语言代码，并与硬件实现的系统在算法运行速度和多通道数据处理能力方面进行了对比，验证了FPGA在传感器数据处理中的优势。

（4） 系统验证与实验 经过实验分析比较，本系统在处理触觉传感器数据时表现出了快速响应和高精度，尤其适合于生活中的碰撞检测场景。我们通过弹力球模拟碰撞，传感器阵列成功感知落点和压力，并通过压力大小预测弹起高度，验证了系统的高精度与快速响应能力。

1.4 关键性能指标

（1） 解调算法FPGA上的实现速度 使用的ADC最大采样速率为200kps。仿真表明，乘法器的计算时间为1个时钟周期。通过将CIC滤波器降采样100倍，每100个输入数据输出一个结果。使用FPGA实现的解调算法可同时处理64路解调通道，达到2kHz的最大读出速率。

（2） 硬件实现性能比较软件实现性能 为直观展现FPGA硬件并行算法的优势，我们在32位Arm Cortex®-M7内核微控制器上编写了相同的64路通道算法，并测试了其处理速度。实验结果表明，FPGA在并行处理能力方面表现出显著优势。使用控制器的定时器外设以微秒为单位计时，在串口接收中观察到的结果为：循环两百次64路串行乘法的结果如绿色为1112us，平均每次64路串行乘法的时间为5.56us，而做均方根运算耗时如红色所示，需要25us。因此运行一次完整的64路算法的时间约为30.56us，求均方根使用数字信号处理库(DSP)加速运算。

（3） 解调信号精度 在得到解调的信号后选取合适长度窗口做平均处理，结果作为传感器的真实值，可以得到该窗口内的信号均方误差。如图所示将砝码放在传感器上得到的信号均方误差。

1.5 主要创新点

（1） 基于离子电子传感器设计的多通道快速响应高精度FPGA读出架构。本作品算法相比于传统的时分复用的读出设计[2]具有低串扰，快速处理等优越性。利用FPGA硬件运算的并行性显著提升了阵列传感器的读出速率，相比软件实现的系统具有全面的优越性，为先进电子皮肤应用提供解决方案。 （2） 设计实际物体碰撞场景，证明本系统可以充分发挥离子电子传感器宽压力度量以及响应灵敏性。在实验中三维动态显示触觉信息，直观展示了高性能电子皮肤的广阔应用前景。 （3） 本系统可灵活调整参数应用于不同的电容阵列传感器，包括变极距型、变面积型和变介质型。具有广泛的应用前景。

第二部分 硬件设计概述

2.1 整理介绍

（1） 系统框图

本系统由电容电压转换器、模数转换器（Analogue to Digital Converter, ADC）、FPGA芯片以及上位机操作系统四个部分组成。电容电压转换器通过运算放大器搭建，可将传感器的电容响应转换为电压信号。ADC部分采用AD7606芯片，具备同步采集8路通道信号的能力。系统的核心是Xilinx-ZYNQ-7020 FPGA芯片，负责驱动ADC进行数据采集和处理，并通过USB接口与上位机进行通信。上位机运行在Windows操作系统下，用于显示和处理采集到的触觉数据。

（2） FPGA芯片数据处理

如示意图所示，在ZYQN-7020芯片的可编程逻辑端（Programmable Logic, PL）实现了ADC驱动、正交参考信号源、数字滤波器和坐标旋转数字计算机（Coordinate Rotation Digital Computer, CORDIC），以及内存共享RAM的读写。在ARM架构的处理器系统端（Processing System, PS），则实现了内存读取和USB通信。相敏验波运算算法通过直接数字信号合成器（Direct Digital Synthesizer, DDS）生成90度相位差的正交参考信号，该信号与采集信号在数字信号处理器（Digital Signal Processor, DSP）中的乘法器相乘，得到互相关函数。随后，信号通过级联积分梳状滤波器（CIC）进行降采样，并通过有限冲激响应滤波器（FIR）滤除高频成分，最终使用CORDIC算法求解传感器响应的幅值。

我们主要编写的Verilog代码实现了ADC芯片的多通道同步数据采集、相敏验波算法以及FIFO缓存发送数据帧的功能。除此之外，还编写了用于调试的逻辑分析仪（ILA）和用于共享数据到PS端的RAM读写模块。数据流向按照图中橙色箭头所示，为简化模块图以突出层次结构，省略了各模块的接口细节。

2.2 各模块介绍

（1） 模数转换器芯片驱动模块 设计模数转换器驱动模块的目的是将模拟信号转换为可处理的数字信号。在芯片初始化时，采样率需设置为至少160 ksps，以满足不同通道信号频率的分辨率要求。考虑到输入信号幅值较小，为保证信号精度，选择了16位的ADC转换精度。所选的ADC芯片支持8通道并行采样，并采用并行方式读取数据，以实现多通道的快速读取。根据激励信号的频率需求，最终确定采样率为200 ksps。

FC-AIMS-tactile-dataset

Dynamic tactile perception based on FC-AIMS platform

Dataset download link: https://drive.google.com/drive/folders/1Ehpyk6XMn0UgFS-sOneyP3ZCYR78MxMv?usp=sharing.

Explanation of hardware circuit

A parallel digital orthogonal demodulator based on FPGA was designed to calculate the capacitance of array nodes under frequency encoding. Multi channel parallel demodulation can meet the high-speed response of sensor arrays. The hardware and digital circuits for frequency encoding and decoding will be organized and open-source in the later stage.

Spatiotemporal Touch Gestures Learning

This project reports a spatiotemporal learning architecture that learns 10 gestures' features using data from our artificial ion mechanoreceptor skin (AIM-skin). We designed 2D convolutional (C2D), 3D convolutional (C3D), and 3D residual convolutional networks (Res3D) respectively to classify and perceive the dynamic tactile sensation of sensors.

System requirements

OS Requirements:

This project is supported for Windows, macOS and Linux and was run on Linux with GPU.

Python Requirements:

The project was compiled using Python (Python 3.9.0) and four NVIDIA T4 tensor core GPUs for model inference.

Installation guide

Some Python packages are required for the compilation.

  1. numpy (version 1.24.3)
  2. torch (version 2.0.0). Torch package provides main APIs to build our dataset, model and training progress.

Instructions to run the project

  1. Please download all data & codes and then unzip all the folders and place them in the same folder. Dataset download link: https://drive.google.com/drive/folders/1Ehpyk6XMn0UgFS-sOneyP3ZCYR78MxMv?usp=sharing.
  2. Run the train.py using Pycharm or other supported softwares or sh do_train.sh. Make sure that you have downloaded all the required packages and used Python with version >=3.8.
    To train the Res3D model, we uses Adam on 4 GPUs with a mini-batch size of 60 examples. The initial learning rate is 0.001 and is divided by 10 every 20 iterations, finishing at 50 iterations (You can modify the parameters).
  3. See the training results on the Validation set. (You may see the results on the test set by running the test.py.)
    Running time for each epoch is close to 10s on our GPU. The whole training progress might take 8 minutes depending on your hardware.

 We recruited 20 participants (14 males and 6 females) to take part in our touch experiments. As an artificial skin, our sensor can capture dynamic tactile information in the experiment. Each subject performed 10 different dynamic touch gestures, with each gesture being recorded four times for data acquisition, and the duration of each touch was 10 seconds. The gestures of hit, stroke, rub, tap, poke, press, scratch, pat, circle, and put were performed by the 20 subjects.
 So, a total of 800 touch gesture samples were collected in the dataset. And each .npz touch data collected from our I-skin has the size of 5000×8×8 because the sensor array is 8×8 and the collecting frequency is 500 frames per second.

Model Description

 An efficient deep 3D Residual ConvNet architecture (shown below) based on the Pytorch framework was constructed for spatiotemporal feature learning, which adopts ResNet as the backbone network by expanding 2D convolutions into 3D convolutions.
(picture of resnet)
 Res3D is composed of multiple basic residual blocks, which include shortcut connections that bypass signals between layers. Inside the residual block are two convolution layers with the size of 3 × 3 × 3, after which batch normalization and ReLU layers are added. This learned tactile output of ResNet can be reviewed as transferable semantic features, forming a new 512D vector, which is used as the input of the fully connected neural network for final decision learning.
